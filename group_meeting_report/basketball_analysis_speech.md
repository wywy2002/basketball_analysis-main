# 篮球视频自动分析系统组会汇报讲稿

## 第1页：篮球视频自动分析系统

这一页主要介绍项目目标。我们的项目是针对篮球比赛视频做自动分析，输入是一段普通比赛视频，输出是带有球员框、篮球轨迹、阵营、球权和战术信息的结果视频。当前项目已经不只是做检测，而是把检测结果继续转化为比赛层面的信息，例如谁控球、是否发生传球或抢断，以及球员在战术板上的位置。最近一项重要改动是把球员阵营识别从大模型方案改成传统视觉方案，降低部署成本，也减少网络和模型加载依赖。

## 第2页：整体处理流程

整体流程可以理解为一条视频分析 pipeline。程序先读取视频并拆成帧，然后用 YOLO 模型分别检测球员、篮球和球场关键点。球员检测结果会交给 ByteTrack，用来在不同帧之间维持同一个球员的 track_id。之后系统会基于球员位置和篮球位置判断球权，再结合阵营识别结果推断传球和抢断。最后把这些分析结果统一绘制回视频，包括球员标注、控球统计、事件统计和 2D 战术板。

## 第3页：核心技术

项目的核心技术主要分成四类。第一是 YOLO 检测，用于识别球员、篮球和球场关键点。第二是 ByteTrack 多目标跟踪，它解决的是同一名球员在连续帧中身份保持的问题。第三是 OpenCV 颜色聚类，这是最近替换大模型后的阵营识别方案，它从球员框中的躯干区域提取球衣颜色，再自动聚成两队。第四是 Homography 坐标变换，用球场关键点把视频画面坐标映射到俯视战术板中，为速度、距离和战术展示提供空间基础。

## 第4页：核心功能

从功能角度看，项目目前已经形成了比较完整的篮球视频分析闭环。基础层面可以检测和跟踪球员、篮球，并把轨迹画在视频上。比赛理解层面可以做阵营识别、球权判断、传球检测和抢断检测。空间分析层面可以把球员映射到 2D 战术板，并估算球员移动距离和速度。最终结果不是单独的数据文件，而是可以直接播放的标注视频，适合复盘、展示和进一步分析。

## 第5页：当前进展与问题处理

这一页重点汇报最近完成的阵营识别改造。之前方案使用 CLIP，根据“白色球衣”“红色球衣”这类 prompt 来判断阵营，效果有一定灵活性，但依赖大模型，也会带来加载慢和部署复杂的问题。现在改成了 YOLO 后处理方案：先用 YOLO 找到球员，再裁剪更可能是球衣的躯干区域，用颜色特征聚类分成两队。针对遮挡和多人重叠的问题，新方案不会强行相信每一帧，而是过滤掉质量差的样本，用跨帧投票和 track 记忆稳定阵营结果。

## 第6页：下一周计划

下一周主要围绕稳定性验证和演示准备推进。首先会用 video_2 等不同视频完整跑流程，观察阵营识别在遮挡、远景和快速移动情况下是否稳定。其次会调优颜色聚类相关参数，比如躯干裁剪比例、颜色混杂阈值和聚类距离阈值。第三会整理更规范的运行方式，尤其是每个视频使用独立缓存，避免不同视频之间串缓存。最后会补充一些可量化结果和典型片段截图，让后续组会汇报更直观。
